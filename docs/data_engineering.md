## 1. Data Source & Acquisition
To construct the *Hukom-AI* corpus, we targeted the **Supreme Court E-Library** (elibrary.judiciary.gov.ph) as the primary source of truth.

* **Total Decisions Scraped:** ~31,000 cases.
* **Target Format:** "Printer Friendly" View.
    * *Rationale:* Standard URLs contained dynamic sidebars, navigation menus, and ads. We reverse-engineered the URL structure to target `/showdocsfriendly/`, which rendered the decisions in a simplified HTML container, reducing noise by approximately 40%.
* **Politeness Protocol:** A strictly enforced rate limit of 2 seconds between requests was implemented to respect server load and prevent IP banning.

## 2. Text Preprocessing Pipeline
Raw legal text is unstructured and noisy. We developed a custom ETL (Extract, Transform, Load) pipeline using Python (`BeautifulSoup` + `Regex`) to normalize the data for Transformer training.

### A. The "Header Guillotine" (De-Identification)
* **Problem:** Legal headers contain the names of the Petitioners, Respondents, and the Division (e.g., "THIRD DIVISION").
* **Risk:** Leaving this data introduces "Leakage." The model might learn to associate specific names (e.g., a famous politician) with a verdict rather than learning the facts.
* **Solution:** We implemented a "Context-Aware Guillotine."
    * The algorithm scans the first 3,000 characters for the Ponente's signature (e.g., `MELO, J.:`, `PER CURIAM:`, `NARVASA, C.J.:`).
    * Upon detection, **everything preceding the name is deleted.**
    * *Fallback:* If no Justice name is found, the algorithm checks for an Uppercase Header ending in a colon (e.g., `DECISION:`) and removes it, provided it does not resemble a time stamp (e.g., `1:00`).

### B. Structural Segmentation (Facts vs. Ruling)
* **Problem:** To train a predictive model, we must isolate the Input (`X` = Facts) from the Target (`Y` = Ruling). Feeding the Ruling into the input would cause 100% accuracy (Data Leakage).
* **Algorithm:** We developed a "Semantic Splitter" that hunts for the Supreme Court's transitional phrases.
    * *Primary Triggers:* "WHEREFORE", "ACCORDINGLY", "FALLO".
    * *Secondary Triggers:* "IN VIEW HEREOF", "IN LIGHT OF THE FOREGOING".
    * *Logic:* The script identifies the *last* occurrence of these triggers (to avoid quoting lower court decisions) and splits the text into two distinct files.

### C. Bias Mitigation (Signatory Removal)
* **Problem:** The "Ruling" section traditionally ends with the names of concurring or dissenting judges (e.g., *"Puno, J., concurs"*).
* **Risk:** "Justice Bias." The model might learn that specific judges are lenient or strict, rather than evaluating the legal merit.
* **Solution:** A "Ruthless Cutter" regex was applied.
    * The script locates the final `SO ORDERED.` phrase.
    * **Action:** It immediately truncates the text at that point. All signatories, dates, and subsequent annexes are permanently deleted.

### D. Noise Reduction
* **Citations:** In-text citations (e.g., `[1]`, `[24]`) were removed using Regex `\[\s*\d+\s*\]` to preserve token space for BERT/RoBERTa (which has a 512-token limit).
* **Footnotes:** Any text appearing after the "Ruthless Cutter" (SO ORDERED) was discarded to prevent footnotes from bleeding into the training data.

## 3. Quality Assurance (Auditing)
We developed a secondary script (`hukom_auditor.py`) to algorithmically verify the integrity of the corpus.

| Check | Logic | Purpose |
| :--- | :--- | :--- |
| **Split Validation** | Checks if file contains `STATUS: SPLIT_SUCCESS` | Ensures Facts and Ruling are separated. |
| **Header Residue** | Scans start of file for "Republic of the Philippines" | Verifies de-identification. |
| **Caps Lock Detection** | Flags files with >80% uppercase in the first 500 chars | Detects headers that survived the guillotine. |
| **Ghost File Detection** | Checks for files < 1KB with "Generated by CMS" text | Removes empty/corrupted downloads. |

* **Total Samples:** 30,918
* **Format:** Structured Text (`URL` | `STATUS` | `FACTS` | `RULING`)
* **Cleanliness Score:** 99.6%
